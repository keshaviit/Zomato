Cart Super Add-On (CSAO) Recommendation System
Problem Understanding
Food delivery platforms today are not only focused on delivering food quickly but also on improving the overall customer ordering experience. Many users place orders by selecting a main dish but often overlook complementary items such as beverages, desserts, or side dishes that could enhance their meals.
The objective of the Cart Super Add-On (CSAO) system is to intelligently recommend relevant add-on options during the ordering process. These recommendations should feel natural and helpful rather than intrusive, while also helping the business increase overall order value.
The system must continuously adapt recommendations based on user context, restaurant information, and operational constraints such as delivery time and pricing.

Dataset Strategy and Assumptions
Since we did not receive an official dataset with the problem statement, we created our own using publicly available restaurant information from Kaggle’s Swiggy restaurant dataset.The dataset includes restaurant-level details like cuisine type, pricing, delivery time estimates, ratings, and location.
However, it does not have real user cart interactions or historical behavior regarding add-on acceptance. To work around this limitation, we made some practical assumptions:We generated synthetic user identifiers to mimic ordering patterns.We derived user behavior signals, like ordering frequency and spending patterns, through aggregation.We simulated add-on acceptance behavior probabilistically, using a mix of restaurant rating, delivery performance, and pricing signals.This approach adds realistic variability and stops the model from learning overly strict rules.



Feature Engineering
Feature engineering focused on capturing signals that influence customer decision-making during food ordering.
User Features
User behavior was approximated using aggregated ordering signals such as total number of orders and average order value. These help represent customer engagement and spending habits.

Restaurant Features
Restaurant quality and performance were captured using average ratings, popularity indicators, cuisine diversity, and delivery efficiency.

Contextual Features
Context plays a major role in recommendations. Geographic location and delivery performance were used as contextual proxies to simulate real-world ordering constraints.

Engineered Features
Additional features were created to better represent user perception of value and convenience:
Price-to-rating ratio to capture perceived value.


Delivery efficiency score combining rating and delivery speed.


Value-for-money metric reflecting quality relative to cost.


These engineered features helped the model capture trade-offs customers often make between price, quality, and convenience.


Modeling Approach
We formulated the recommendation problem as a contextual ranking task.
Instead of directly predicting which item a user will order, the model estimates the probability that a user will accept a recommendation given available contextual information.
A LightGBM Gradient Boosting model was selected due to its strong performance on structured tabular datasets and its ability to model complex feature interactions efficiently.
Recommendations are generated by ranking candidate restaurants based on predicted acceptance probability.

Real-Time Recommendation Architecture
The proposed system follows a simple production-inspired architecture.
First, candidate restaurants are filtered based on location and price compatibility.
Next, the ranking model evaluates each candidate using contextual features and assigns an acceptance probability score.
Finally, recommendations are sorted and the top suggestions are displayed to the user.
This approach supports fast inference and can be extended for real-time deployment with feature caching.

Evaluation Framework
Model performance was evaluated using ranking-oriented metrics.
AUC was used to evaluate overall discrimination ability between accepted and rejected recommendations.
Precision@10 measures the quality of top recommendations shown to users.
NDCG@10 evaluates ranking quality by rewarding correct placement of relevant suggestions higher in the recommendation list.
These metrics align closely with real CSAO rail interaction behavior where users primarily engage with top-ranked items.

Business Impact
The system focuses on balancing customer experience and business growth.
Relevant add-on suggestions can improve average order value while maintaining user trust.
Based on simulation results, the system demonstrates potential to improve add-on attach rates and achieve an estimated 5–12% uplift in cart value without negatively impacting ordering flow.

Limitations and Future Work
The primary limitation of this work is the absence of real cart-level transactional data.
Acceptance labels were simulated using probabilistic assumptions rather than observed behavior.
Future improvements could include:
Item-level co-occurrence modeling.


Sequential recommendation models.


Reinforcement learning based optimization.


Real-world A/B testing for validation.



Model Development Summary
The solution was implemented using Python with Pandas, NumPy, Scikit-learn, and LightGBM.
The pipeline included:
Data preprocessing and aggregation.


Feature engineering.


Probabilistic label simulation.


Model training using stratified train-test split.


Offline evaluation using ranking metrics.
